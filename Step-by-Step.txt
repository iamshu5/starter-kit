Step-by-Step: Tambah Halaman Baru & Menu/Sub Menu
(Panduan lengkap + kode lengkap ada di GUIDE-ADD-FEATURE.md)
(Dokumentasi arsitektur keseluruhan ada di APP-FLOW.md)

=== BACKEND ===

1. Migration
   - Buat file di database/migrations/
   - Kolom yang di-search → masukkan fullText()
   - Kolom yang di-WHERE/ORDER → pasang ->index()

2. Model
   - fillable, casts, relasi (belongsTo, dll)
   - Jika ada file: accessor getXxxUrlAttribute() + $appends

3. Service (app/Services/ProductService.php)
   - use FullTextSearch trait
   - paginate / create / update / delete

4. Form Requests (StoreProductRequest + UpdateProductRequest)
   - Store: 'required'; Update: 'sometimes'
   - Update unique: "unique:products,slug,{$productId}"

5. Controller (app/Http/Controllers/Api/ProductController.php)
   Semua method WAJIB dibungkus ApiResponse::tryCatch():
   
   public function store(StoreProductRequest $request, ProductService $service): JsonResponse
   {
       return ApiResponse::tryCatch(
           fn() => ApiResponse::success($service->create($request->validated()), 'Created', 201)
       );
   }

6. Routes (routes/api.php) — dalam middleware('auth:sanctum'):
   Route::prefix('/products')->group(function () {
       Route::get('/',            [ProductController::class, 'index'])  ->middleware('permission:products.view');
       Route::get('/{product}',   [ProductController::class, 'show'])   ->middleware('permission:products.view');
       Route::post('/',           [ProductController::class, 'store'])  ->middleware('permission:products.create');
       Route::put('/{product}',   [ProductController::class, 'update']) ->middleware('permission:products.update');
       Route::patch('/{product}', [ProductController::class, 'update']) ->middleware('permission:products.update');
       Route::delete('/{product}',[ProductController::class, 'destroy'])->middleware('permission:products.delete');
   });

7. Seeder
   - Tambah permissions: products.view/create/update/delete
   - Tambah menu entry
   - Jalankan: php artisan migrate:fresh --seed

=== FRONTEND ===

8. services/api/products.js
   export const productsApi = { list, get, create, update, remove }

9. features/products/components/ProductForm.jsx
   - react-hook-form; Toggle WAJIB pakai Controller (bukan register)

10. features/products/pages/ProductsPage.jsx
    - useQuery + 3 useMutation (create/update/delete)
    - Import FormModal + ConfirmDeleteModal dari @/components/ui/ (REUSABLE — jangan buat per-fitur)
    - <FormModal modal={modal} entityLabel="Product" onClose={...}><ProductForm /></FormModal>
    - <ConfirmDeleteModal open={!!deleting} title="Delete Product" onClose={...} onConfirm={...} isLoading={...}>
    - onError: (e) => toast.error(toastMsg(e))

11. Daftarkan di App.jsx (lazy import + Route dalam PermittedRoute)

12. Tambah judul di AppLayout.jsx:
    const titles = { ..., '/products': 'Products' }

=== TAMBAH MENU VIA ADMIN PANEL ===

Login → halaman Menus → Add Menu:
  Name: Products | Slug: products | Route: /products
  Order: angka urut | Icon: SVG dari heroicons.com
  Parent: Top Level (atau pilih parent untuk sub-menu)

Roles → ikon ⚙ → tab Menus → centang menu baru → Save Menus
→ User dengan role tersebut otomatis bisa akses /products di sidebar

=== JIKA ADA FILE UPLOAD ===

Lihat GUIDE-ADD-FEATURE.md Bagian 2 (Upload Step A–H).

SINKRONISASI WAJIB — ketiga nilai ini harus SAMA:
  max:5120         di FormRequest
  maxKb: 5120      di FileService (wajib eksplisit!)
  maxKb={5120}     di FileInput component

Jenis file juga harus konsisten:
  mimes:jpeg,png,webp            di FormRequest
  ['image/jpeg','image/png',...]  di FileService allowed
  accept="image/jpeg,..."        di FileInput

Accessor di Model: app(FileService::class)->url($this->xxx_path)
